NEW ARCHITECTURE
================================================================================

Container Composition:
  PostgreSQL 14 → /mnt/user/appdata/pdf-lm/postgres
  Redis 7 → /mnt/user/appdata/pdf-lm/redis
  pdf-lm (FastAPI API port 8000 + Celery worker via Supervisor)
  React Frontend (port 3000)

Benefits:
  • Reduced memory overhead (shared Python environment)
  • Simpler image management (one image instead of two)
  • Native process supervision with auto-restart
  • Better log aggregation to persistent volumes
  • Easier Unraid UI integration

================================================================================
QUICK DEPLOYMENT (Unraid)
================================================================================

1. Create directories:
   mkdir -p /mnt/user/appdata/pdf-lm/{postgres,redis,uploads,logs}
   mkdir -p /mnt/user/PDFGameVault

2. Generate credentials:
   openssl rand -base64 32  (DB password)
   openssl rand -hex 32     (Secret key)

3. Create .env file at /mnt/user/appdata/pdf-lm/.env with credentials

4. Download docker-compose.prod.yml from repository

5. Deploy:
   cd /mnt/user/appdata/pdf-lm
   source .env
   docker-compose -f docker-compose.prod.yml up -d

6. Access: http://YOUR_IP:3000

Full details in UNRAID_INSTALL.md or UNRAID_QUICK_START.md

TESTING CHECKLIST
================================================================================

Before production use:
  - Verify container startup: docker-compose ps (all "Up" + healthy)
  - API health: curl http://localhost:8000/health
  - Supervisor status: docker exec pdf-lm-backend supervisorctl status
  - Database: docker exec pdf-lm-postgres psql -U pdf_user -d pdf_library -c "\dt"
  - Web UI: Open http://localhost:3000, register user, login
  - Library scan: Admin panel → "Scan Library"
  - PDF import: Place PDFs in /mnt/user/PDFGameVault/, rescan
